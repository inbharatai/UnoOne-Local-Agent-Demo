# Contributing to UnoOne Public Demo

Thank you for your interest! This public repo is open for contributions in specific, safe areas.

## What We Welcome

- **UI improvements** — Compose screens, animations, theme refinements
- **Android shell enhancements** — Navigation, ViewModels, lifecycle fixes
- **Generic STT/TTS adapters** — New adapter interfaces for alternative local engines
- **Documentation** — Fixes, translations, clearer setup instructions
- **Demo workflows** — Example skills, test scripts, sample commands
- **Bug fixes** — In the public shell or generic modules

## What We Do NOT Accept

- Proprietary AI layer implementations (prompts, routing, model tuning)
- Internal tool names or proprietary action handlers
- Real API keys, credentials, or service account files
- Private datasets or user data
- Attempts to reverse-engineer excluded proprietary logic

## How to Contribute

1. **Fork** the public repository.
2. **Branch** from `main`.
3. **Make your changes** within the allowed scope above.
4. **Test** on a real Android device or emulator.
5. **Open a Pull Request** with a clear description of what changed and why.

## Code Style

- Kotlin with Android conventions
- Compose UI follows Material 3 guidelines
- Keep comments neutral — no promotional language
- Mark any stub/demo code clearly with `// DEMO:` or `// STUB:` comments

## Questions?

Open a Discussion (not an issue) for questions about architecture or contribution scope.
